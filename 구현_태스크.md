# 구현 태스크 (빌드 순서)

이 순서대로 구현한다. 각 단계 완료 후 실제 동작을 확인하고 다음으로 넘어간다.

## 0. 프로젝트 셋업
- [ ] `npm init`, Node 18+ 확인, `"type": "module"` 설정.
- [ ] 의존성 설치: `express`, `@supabase/supabase-js`, `cors`, `dotenv`.
- [ ] 폴더 구조 생성: `src/`, `public/`, `data/`, `outputs/`.
- [ ] `.env.example` 작성(키 이름만, 값 비움).
- [ ] `.gitignore`: `.env`, `outputs/`, `data/voice_profile*.json`.
- [ ] `package.json` scripts: `"start": "node src/server.js"`, `"dev": "node --watch src/server.js"`.

## 1. Anthropic 래퍼 (`src/anthropic.js`)
- [ ] `callClaude({ model, systemPrompt, userPrompt, maxTokens, cacheTranscript })` 구현.
- [ ] `cacheTranscript` 옵션 true 시 userPrompt 내 자막 블록에 `cache_control: {"type": "ephemeral"}` 적용.
- [ ] 응답에서 text 블록만 모아 반환. 실패 시 상태코드·단계 포함 에러.

## 2. Vimeo 자막 수집 (`src/vimeo.js`)
- [ ] `parseVideoId(url)` — `vimeo.com/{id}`, `player.vimeo.com/video/{id}`, `vimeo.com/{id}/{hash}`, 공유 링크(`vimeo.com/share/{hash}`) 형태 대응. 공유 링크는 Vimeo oEmbed API로 video_id 추출.
- [ ] `getTextTracks(id)` — `GET /videos/{id}/texttracks`, `Authorization: bearer` 헤더 포함.
- [ ] 한국어 트랙 우선, 없으면 첫 트랙, 없으면 명확한 에러.
- [ ] VTT fetch 시에도 `Authorization: bearer` 헤더 포함.
- [ ] WebVTT 파싱: HTML 태그(`<b>`, `<c.color>` 등) 제거, `[MM:SS] 텍스트` 조립.
- [ ] 자막 압축: 큐가 많으면(60분 초과 기준) 30초 간격 샘플링 압축본 생성. `{ full, compressed }` 반환.
- [ ] `getTranscript(url)` 로 위 과정 하나로 노출.
- [ ] 검증: `vimeo.com/1190465798` 로 자막이 뽑히는지 확인.

## 3. Instagram 캡션 수집 (`src/instagram.js`)
- [ ] API 버전 상수 `const GRAPH_API_VERSION = 'v21.0'` 파일 상단 정의.
- [ ] `fetchCaptions({ limit })` — 그래프 API로 캡션 수집, 빈 캡션 제외.
- [ ] 페이지네이션으로 30~50개 확보.
- [ ] 401 에러 시 "Instagram 액세스 토큰이 만료됐습니다. 핸드북 프로젝트의 갱신 스크립트를 실행하세요" 안내.
- [ ] 검증: 실제 계정에서 캡션 수집 확인.

## 4. 보이스 프로파일 (`src/voice.js`)
- [ ] `learn()` — 캡션 수집 → 말투 분석 프롬프트 → 저장 전 기존 파일 `voice_profile_YYYYMMDD.json` 백업 → `data/voice_profile.json` 저장 + Supabase `voice_profile` 테이블 upsert.
- [ ] `load()` — Supabase 우선, 없으면 로컬 파일. 둘 다 없으면 `learn-voice` 안내 후 종료.
- [ ] 검증: `learn()` 실행 후 프로파일이 자야쌤 문체를 잘 담았는지 사람이 확인.

## 5. 프롬프트 조립 (`src/prompts.js`)
- [ ] `프롬프트_템플릿.md`의 5종을 함수로 구현(치환 변수 주입).
- [ ] cafe·caption에는 VOICE/EXAMPLES 주입, capture·carousel에는 자막 위주.
- [ ] `buildRevisePrompt(type, originalContent, instruction, voice, transcript)` — 수정 재생성용 프롬프트.
  - 원본 내용 + 수정 지시 + 보이스 + 자막 기반으로 해당 섹션만 재생성.
- [ ] `buildRegeneratePrompt(type, approvedCafe, approvedCaption, transcript)` — 캐러셀·캡쳐 재생성용.
  - 확정된 카페/캡션 내용을 컨텍스트로 포함해 일관성 확보.

## 6. DB 클라이언트 (`src/db.js`)
- [ ] Supabase 클라이언트 초기화.
- [ ] `saveJob(jobData)`, `saveContent(jobId, type, content)`, `getJob(id)`, `getContents(jobId)`.
- [ ] 로컬 개발 시 Supabase 미연결이면 파일 시스템 폴백(`outputs/<video_id>/`).

## 7. 생성 오케스트레이션 (`src/generate.js`)
- [ ] `generateAll({ url, topic, onProgress })` — 4종 순차 생성.
  - `onProgress(type, content)` 콜백으로 각 섹션 완료 시 즉시 알림.
  - 순서: cafe → caption → carousel → capture.
  - cafe·caption은 compressed 자막, capture는 full 자막 사용.
  - 자막 블록에 `cache_control` 캐싱 적용.
- [ ] `revise({ jobId, type, instruction })` — 카페 또는 캡션 수정 재생성.
- [ ] `regenerateSecondary({ jobId })` — 캐러셀·캡쳐 재생성 (확정된 cafe·caption 기반).
- [ ] 실패한 섹션만 재시도 가능하게 구조화.

## 8. Express 서버 (`src/server.js`)
- [ ] `public/` 정적 파일 서빙.
- [ ] **POST `/api/generate`** — URL·주제 입력 → 자막 수집 → SSE로 섹션별 완료 이벤트 스트리밍.
  ```
  event: progress
  data: { type: 'cafe', content: '...' }
  
  event: progress
  data: { type: 'caption', content: '...' }
  ...
  event: done
  data: { jobId: '...' }
  ```
- [ ] **POST `/api/revise`** — `{ jobId, type, instruction }` → 재생성 결과 반환.
- [ ] **POST `/api/regenerate`** — `{ jobId }` → 캐러셀·캡쳐 재생성 결과 반환.
- [ ] **POST `/api/learn-voice`** — 보이스 프로파일 갱신.
- [ ] **GET `/api/job/:id`** — 저장된 결과 조회.
- [ ] 에러 응답 형식 통일: `{ error: '단계명: 원인 메시지' }`.

## 9. 프론트엔드 (`public/`)
- [ ] **`index.html`** — URL 입력 폼, 강의 주제 입력, [생성 시작] 버튼.
- [ ] **`app.js`** — SSE 연결로 섹션별 결과 수신 → 화면 순차 표시.
  - 각 섹션: 텍스트 영역 + [복사] 버튼 + 수정 요청 입력란 + [재생성] 버튼.
  - 카페·캡션 둘 다 표시된 후 [캐러셀·캡쳐 가이드 재생성] 버튼 활성화.
  - 생성 중인 섹션 스피너 표시.
- [ ] **`style.css`** — 깔끔하고 모바일 대응되는 레이아웃.
- [ ] Vanilla JS만 사용 (프레임워크 없음).

## 10. 통합 검증
- [ ] 실제 강의 URL로 전체 플로우 실행: 생성 → 수정 요청 → 재생성 → 캐러셀·캡쳐 재생성.
- [ ] 카페·캡션이 자야쌤 말투와 일치하는지 사람이 점검.
- [ ] 캡쳐 가이드 타임스탬프가 실제 영상 시점과 맞는지 확인.
- [ ] 글자수 기준 충족 확인 (카페 2,000자 / 캡션 1,000~1,300자).

## 11. 배포 준비 (로컬 검증 후)
- [ ] Supabase DB 테이블 생성 (jobs, contents, voice_profile).
- [ ] Railway 또는 Render에 Express 서버 배포.
- [ ] 환경변수 설정 (SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY 등).
- [ ] 도메인 또는 고정 URL 확인 후 팀 공유.

## 확장 과제 (2단계)
- 자막 없는 영상용 Whisper STT 폴백.
- 생성 이력 조회 페이지 (영상 목록 + 결과 재열람).
- 여러 영상 일괄 처리.
- 구글 독스 내보내기.

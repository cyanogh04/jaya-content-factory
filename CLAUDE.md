# CLAUDE.md

이 파일은 클로드 코드가 이 프로젝트에서 작업할 때 항상 따르는 지침이다.

## 프로젝트
비메오 강의 URL → 자막 자동 수집 → 자야쌤 말투(인스타 캡션 학습) 기반으로 카페 서머리·인스타 캡션·캐러셀 기획·캡쳐 가이드 4종을 생성하는 **웹앱** (Node.js Express + Vanilla JS 프론트엔드). 브라우저에서 URL 입력 → 섹션별 순차 생성 → 수정 요청으로 재생성 가능. 상세는 `기획서_PRD.md`, 프롬프트는 `프롬프트_템플릿.md`, 빌드 순서는 `구현_태스크.md` 참조.

## 자야쌤 · 브랜드 컨텍스트
- 필명 '자야쌤', 15년차 소잉 교육/콘텐츠 사업. 브랜드: 소잉클래스 + 디자인드바이.
- 대상 독자: 제대로 만들고 싶은 30~50대 취미 소잉러.
- 철학: "옷을 만드는 게 아니라 짓는다." '왜 이렇게 하는지'까지 담는다.
- 어조: 따뜻하고 분명한 구어체. 말줄임표(…)와 부드러운 끝맺음, 직접 말 거는 화법.
- 목적: 진정성 있는 브랜딩으로 수강·공구 매출 성장. 모든 산출물은 프로페셔널해야 한다.
- 브랜드 컬러(캐러셀 기획 참조용): 베이지 `#fff4e1` 기본 / 진네이비 `#18284e` 포인트. 컨셉은 모던·깔끔·노블, 실사 사진/사실적 일러스트(아기자기 지양).

## 절대 규칙 (지키지 못하면 작업 중단)
1. **말투는 반드시 `data/voice_profile.json` 기준**으로 생성한다. 프로파일이 없으면 먼저 `learn-voice` 실행을 안내한다.
2. **자막에 없는 사실을 지어내지 않는다.** 캡쳐 타임스탬프는 자막의 실제 시각에서만 가져온다.
3. **입력은 비메오 URL 하나**로 끝나야 한다. 사용자에게 자막을 직접 붙여넣게 하지 않는다.
4. 캐러셀 후크 구조는 **좌절/실패로 시작 → 공감 → 자야쌤의 해법(전문성) → CTA**를 따른다.
5. 시크릿(API 키/토큰)은 `.env`에서만 읽고, 코드·로그·깃에 절대 남기지 않는다. `.gitignore`에 `.env` 포함.
6. 이미지/카드뉴스 생성은 이 프로젝트 범위가 아니다(텍스트 초안까지만).

## 스택 / 실행
- Node.js 18+ (내장 fetch), Express, @supabase/supabase-js, Vanilla JS 프론트엔드.
- 실행: `npm run dev` (로컬, http://localhost:3000).
- 로컬 개발 시 Supabase 미연결이면 파일 시스템 폴백 자동 적용.

## 환경변수
`ANTHROPIC_API_KEY`, `VIMEO_ACCESS_TOKEN`, `INSTAGRAM_ACCESS_TOKEN`, `IG_USER_ID`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `PORT`

## 모델 선택
- caption, cafe_summary → `claude-opus-4-8` (말투 중요).
- capture_guide, carousel_plan → `claude-sonnet-4-6` (구조 위주, 비용 절감).
- 한국어 출력이 기니 `max_tokens`는 4000 이상.

## 코딩 컨벤션
- 프롬프트 문자열은 `src/prompts.js` 한 곳에 모으고, 코드 로직과 섞지 않는다.
- 각 외부 연동(vimeo/instagram/anthropic)은 독립 모듈로 분리, 각자 에러 메시지를 명확히 던진다.
- 콘솔 로그는 단계별로 무엇을 하는지 한국어로 안내(예: "자막 12개 큐 수집 완료").
- 네트워크 호출은 실패 시 원인(상태코드·단계)을 사용자에게 보여준다.

## 50만뷰+ 트렌드 반영에 대한 메모
캐러셀 기획 로직에는 자야쌤의 검증된 후크 구조를 내장한다. 다만 매 실행마다 50만뷰+ 레퍼런스를 실시간 분석하는 것은 이 자동화 범위 밖이며, 분기별로 사람이 레퍼런스를 분석해 프롬프트의 기획 틀을 갱신하는 방식으로 운용한다.
